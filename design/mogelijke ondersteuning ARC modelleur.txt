Welke ondersteuning willen we de modelleur in ARC geven?

1. Duidelijke aanwijzingen als hij een fout heeft gemaakt.
	a. Syntactische fouten, zie de opsomming van mogelijke expressies.
	b. Semantische fouten:
		- rolvulling met een niet bestaande rol
		- rolvulling die niet past bij UserRol
		- rolvulling die niet past bij Actor van Actie
		- inbedding van een verkeerd soort context (b.v. Domein in Zaak)
		- niet bestaande property in een view opgenomen
		- property in een view opgenomen die niet bij de rol hoort
		
2. Hints over wat zijn vervolgstap kan zijn. Dat gaat om:
	- (na een voltooide expressie) wat kan de volgende expressie zijn? Zie Opeenvolging Van Expressies.
	- (als een verwijzing ingevuld moet worden) wat is een legale verwijzing?


Een overzicht van wat bij andere talen zoal wordt geboden:
	- code completion. Dit dekken we deels met snippets. Maar dat is puur syntactisch, terwijl semantische completion ook kan.
	- code formatting. Dat hebben we met syntax coloring en autoindent geregeld.
	- highlight occurrences. Ace ondersteunt dat out of the box voor geselecteerde woorden.
	- renaming. Dat gaan we ondersteunen.
	- outline view
	- jump to definition
	- quick fixes and quick suggestions (b.v. de suggestie om een property te definiÃ«ren als je in een view naar een onbekende property verwijst).
	
Min of meer onafhankelijk van het soort ondersteuning is de kwestie: via welk kanaal? We hebben in Ace de volgende mogelijkheden:
- via een tooltip in de linkermarge
- met autocomplete
- met een tooltip boven een woord
- vermoedelijk

Foutdetectie bestaat altijd uit twee stappen: 
wat voor soort expressie probeert de modelleur te schrijven
welke fout maakt hij daarin?
Als (1) niet gedetermineerd is, is de beantwoording van (2) lastig.

OPEENVOLGING VAN EXPRESSIES
Op basis van een syntactische typering van een expressie kunnen we de mogelijkheden voor de volgende expressie sterk beperken.

OP EEN:										VOLGT EEN:
Context declaratie							Sectie
Properties Sectie							Scope declaratie
Rollen Sectie								Rol declaratie
Views Sectie								View declaratie
Acties Sectie								Actie
Scope declaratie							Property declaratie
Rol declaratie met ingebedde definitie		Context declaratie of Query
View declaratie								property naam
PropertyQuery								Query 

Er zijn dus drie expressies waarop meerdere andere soorten expressies kunnen volgen:
	- context declaratie: drie soorten secties, of: vijf soorten context declaratie.
	- properties sectie: intern of extern
	- rol declaratie met ingebedde definitie: context declaratie of query.
Er zijn vijf expressies waarop dezelfde soort expressie weer kan volgen:
	- context declaratie
	- property declaratie
	- view declaratie
	- rol declaratie
	- actie

VERWIJZINGEN
Property verwijzingen. In een view kan alleen verwezen worden naar properties die bij dezelfde rol zijn gedefinieerd als waar de view zelf is gedefinieerd.
Rolvulling. Een rol mag gevuld worden met een ongekwalificeerde naam als die in scope is, of met een gekwalificeerde naam.
