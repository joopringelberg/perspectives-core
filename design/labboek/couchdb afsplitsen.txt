COUCHDB AFSPLITSEN
Het lijkt erop dat dit goed gelukt is. Maar ik wil het testen met de clients die ik heb.
Hoe doe ik dat?

1. Welke clients heb ik om te testen?
  - perspectives-react-integrated-client (intern kanaal)
  - perspectives-development-client (TCP)
2. Hoe zorg ik ervoor dat deze clients de branch van de core gebruiken?
3. Hoe gebruiken deze clients de core? (hoe is de dependency op de core)
  - perspectives-development-client gebruikt de core alleen over TCP. Dus dit project kan ik direct gebruiken om te testen!
  - perspectives-react-integrated-client gebruikt de core via package.json, dus die wordt met npm ge√Ønstalleerd.
  Maar ik heb de directory in node_modules vervangen door een symbolic link naar het core project. Ik hoef daarom
  niets te veranderen in het gebruikende project zelf!
4. Test eerst de integrated client. Als die de Core draait, kan ik de development client draaien over TCP.

DE INTEGRATED CLIENT TESTEN

npm run electron

CONCLUSIE
De afsplitsing van Couchdb is geslaagd. Immers, alleen de Core gebruikt Couchdb. Die heb ik getest via de integrated client.
Het is niet nodig hem ook nog te testen via de remote client.

Bij nader inzien: het is de vraag of dat zo is, of dat ik een oude build heb gebruikt! Dat klopte, maar na compilatie
van de core deed ie het ook nog steeds.
