De situatie is dat we in een branch zitten en dat op de master commits zijn gedaan (die ik kan missen). Ik wil terug naar de situatie dat de master wijst naar de huidige laatste commit van de perspectives-electron branch.

In de eerste plaats heb ik de master uitgecheckt.
Vervolgens de master teruggezet naar de ancestor van de master en de ontwikkelingsbranch, perspectives-electron:

joopMac-2:perspectives-core joopringelberg$ git checkout master
Switched to branch 'master'
Your branch is up-to-date with 'origin/master'.
joopMac-2:perspectives-core joopringelberg$ git reset --hard 3489dd6
HEAD is now at 3489dd6 Filter query combinator

Nu is de master een directe ancestor van perspectives-electron. Als ik een merge doe, wordt een fast forward uitgevoerd:

joopMac-2:perspectives-core joopringelberg$ git merge perspectives-electron
Updating 3489dd6..8e84854
Fast-forward
 .eslintrc.json                                                      |   36 +
 .gitignore                                                          |   44 +
...

Nu kan ik perspectives-electron weggooien, want die wijst naar dezelfde commit als master.

joopMac-2:perspectives-core joopringelberg$ git branch -d perspectives-electron
Deleted branch perspectives-electron (was 8e84854).
joopMac-2:perspectives-core joopringelberg$ git branch
* master

Ik zit nu op de master, geen branches meer die nog moeten mergen.
