ACTION
Dit is de huidige structuur in de file:

	Action : Consults : Consults
		ActionOnScreen : False						/// Is dit niet redundant? Het scherm bevat een verwijzing
		ActionObjectView : AllProperties
		Role : ObjectRole : OR of Consults
			Object : Feest
		Role : SubjectRole : SR of Consults
			Subject : Host

1. In plaats van Role... zou je gewoon Object kunnen gebruiken (geeft evenveel informatie en het zijn eigenlijk geen rollen):

	Object : Feest

Je kan dan weliswaar geen naam van de Object 'rol' aangeven, maar dat doe je toch niet.

2. Als je dan toch Role... gebruikt, houd dan het stramien aan dat je ook voor rollen van een context gebruikt. Dus:

		Role : ObjectRole : OR of Consults
			FilledBy : Feest
3. De view volgt ook een heel ander patroon dan bij rol:

	ActionObjectView : <naam>

in plaats van:

	View : Object : <naam>

Alles bij elkaar zou het dit kunnen worden:

	Action : Consults : <naam>
		View : Object : <ref>							/// optioneel: indien afwezig de view van het perspective
		Role : Object : <ref>
		IndirectObject : <ref>
		View : IndirectObject : <ref>

of, nog regelmatiger:

	Action : Consults : <naam>							/// hoewel je nooit deze naam gebruikt, als modelleur
		Role : Subject : <ref>
			View : ViewRef : <ref>						/// Of de View inbedden? 
		Role : Object : <ref>
			View : ViewRef : <ref>

Hoort een view exclusief bij een Action, verwijs je naar een view van de Role?
