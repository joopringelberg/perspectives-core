een grammatica:

contextDefinition = type identifier BLOCK (rolDefinition | privatePropertyDefinition | publicPropertyDefinition | RolAssignment)*
rolDefinition = type identifier BLOCK (propertyDefinition)*
propertyDefinition = ':PropertyType' identifier BLOCK propertyAssignment*
privatePropertyDefinition = 'private' BLOCK propertyAssignment*
publicPropertyDefinition = 'public' BLOCK propertyAssignment*


Is propertyDefinition een specialisatie van context of contextDefinition?
Ja: propertyDefinition is context, als type = ':PropertyType' en er geen rolAssignments zijn.

Hebben we die specialisatie nodig? 
De beperking dat een PropertyDefinition geen rolAssignments heeft, is semantisch. Dat dwingen we dus op een andere manier af.
Kortom: nee, we hebben propertyDefinition niet nodig. Dus:
rolDefinition = type identifier BLOCK (context)*


Is rolDefinition een specialisatie van context?

Nee.

Dan is dit de volledige grammatica:

file = expr*
expr = contextDefinition | context
contextDefinition = type identifier BLOCK 
	( rolDefinition 
	| privatePropertyDefinition 
	| publicPropertyDefinition 
	| RolAssignment
	)*
rolDefinition = type identifier BLOCK context*
privatePropertyDefinition = 'private' BLOCK propertyAssignment*
publicPropertyDefinition = 'public' BLOCK propertyAssignment*
rolAssignment = type '=>' identifier BLOCK propertyAssignment*
propertyAssignment = type '=' simpleValue
simpleValue = 
	  string
