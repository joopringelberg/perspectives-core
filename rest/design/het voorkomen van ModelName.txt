In een contextdeclaration met type C, kan D dan een ModelName zijn? Of alleen een QualifiedName?
Dus: C D, bijvoorbeeld: psp:Property <declaredName>
Voor C is:
	psp:Context					ja
	psp:Property				nee
	psp:Rol						nee
	psp:PropertyReferentie		nee
	psp:view					nee
Conclusie: alléén een psp:Context kan een ModelName dragen. Oftewel, als iets een ModelName als PerspectRI heeft, is het pspType daarvan psp:Context. In alle andere gevallen moet het een QualifiedName zijn.

Type C moet altijd een QualifiedName zijn, want we maken geen instanties van een Model.
